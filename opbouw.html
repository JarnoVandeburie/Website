Gebruikte technieken

HTML5 en CSS

De pagina is gebouwd met HTML5 en CSS met in totaal 4 HTML-files: Home, alle artikelen, info over een artikel en de opbouw van de website.
Doordat de artikelen dynamisch worden ingeladen, is er geen nood om voor elk artikel een nieuwe HTML-file aan te maken.

De CSS wordt altijd gereset met behulp van een bestandje genaamd reset.css. Dit is een universeel bestand en maakt het gemakkelijker om stijlen aan te passen zonder te botsen op de automatische stijl die gegenereerd wordt.

JavaScript

Het navigatiemenu wordt gegenereerd aan de hand van JavaScript. Dit is gedaan zodat het makkelijker is om nieuwe paginas toe te voegen. Zo moet er niet in elke HTML-file de header worden aangepast.

Ook het genereren van de artikelen gebeurt aan de hand van JavaScript. De artikelen worden gelezen uit een file waar ze opgeslagen staan in een JSON-formaat. De code overleest elk element en plaatst deze op de juiste plaats in het artikel. De afbeeldingen worden ook uit de file uitgelezen en dan uit een folder gehaald.
Een kleine extra is dat er ook een optie is om de artikelen compact weer te geven. Deze knop geeft alle artikelen een klasse waardoor sommige elementen verborgen worden.

Om het juiste artikel weer te geven als je op een titel klikt, wordt de titel als parameter meegegeven in de URL en wordt dan in de JSON naar het juiste artikel gezocht. Zodat de parameter wat leesbaarder zou worden voor de eindgebruiker, worden alle spaties vervangen door een tilde. Dit zorgt er voor dat dit teken niet mag gebruikt worden in de titel van een artikel, maar dat leek mij geen probleem.

JSON

Zoals aangehaald in het hoofdstukje hierboven, wordt er gebruik gemaakt van JSON om de artikelen in te lezen. Alle belangrijke elementen hebben hun eigen parameters. Er is ook een optie voor een beschrijving toe te voegen aan de artikelen. Dit is een samenvatting van het artikel en dit wordt alleen weergegeven op het scherm met alle artikelen.

Waarom op deze manier?
Ik heb gekozen voor een JSON-bestand omdat ik een manier wou kiezen die ook eenvoudig is voor niet-ITers. Als iemand mij zou vragen hoe zij het best dit kunnen implementeren op hun webiste, dan moeten ze geen HTML leren maar kunnen ze gewoon de tekst in het JSON-bestand toevoegen.

Hosting

Deze website is online toegankelijk met behulp van GitHub. Nadat ik een deel van de site heb toegevoegd of bijgewerkt stuur ik de code naar mijn GitHub repository. GitHub update dan automatisch de getoonde website.
Tijdens het bewerken gebruikte ik py -m http.server om lokaal een website te hosten. Zo moest ik niet na elke kleine aanpassing de code op GitHub plaatsen om de veranderingen te zien. Dit was nodig omdat ik de artikelen anders niet kon lezen.

Nieuw artikel toevoegen

Tekst toevoegen

Het toevoegen van tekst aan een artikel is heel simpel.
Doordat er gebruikt gemaakt wordt van JSON, kan een nieuw artikel toegevoegd worden in artikels.json door middel van het dupliceren van een vorig item en daarna de eigenschappen in te vullen.

Afbeeldingen toevoegen

Alle gebruikte afbeeldingen in verband met de artikelen moeten in de folder assets/images/articles komen. Elk artikel heeft daar een eigen mapje.
Om het logo te veranderen moet de naam van het bestand (zonder extensie) meegegeven worden als waarde voor logo in de JSON
Om een afbeelding in de tekst te plaatsen maak je een nieuwe string aan die begint met @IMAGE- gevolgd door de bestandsnaam zonder extensie. Bijvoorbeeld: @IMAGE-myCat

Een lijst toevoegen

Een lijst toevoegen is super simpel. Net zoals bij de afbeeldingen is er een bepaald voorvoegsel dat je kan gebruiken. Voor een lijst is dit natuurlijk @LIST-.
Items binnen in de lijst kan je splitsen door een tilde (~) met een spatie er voor en er na te gebruiken.
Bijvoorbeeld: @LIST-cat ~ dog ~ monkey ~ pig ~ lion
